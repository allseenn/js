# 01.Intro

[01.Lecture](https://github.com/allseenn/js/tree/main/01.Lecture)

## О JavaScript

**HTML** — язык разметки для веб‑страниц.

**CSS** — язык оформления (стили) для HTML.

**JS** — язык программирования.

JavaScript как язык состоит из двух логических частей:

- ECMAScript — спецификация языка (синтаксис, типы, поведение).
- Web/API — набор API, которые предоставляет среда выполнения (например, браузер или Node.js).

## Движки

Список популярных движков:

- V8 (Chrome, Node.js)
- SpiderMonkey (Firefox)
- JavaScriptCore (Safari)
- Chakra (старые версии Edge)
- Rhino (на JVM)

Движки, как правило, написаны на C/C++ и реализуют парсинг, интерпретацию/JIT и сборку мусора.

## Среды выполнения

- Браузеры (включают HTML/CSS/JS парсеры, Rendering Engine, Network Stack и т.д.).
- Node.js, Electron, Deno, React Native.
- Встраиваемые системы (приборы, роботы и т.п.).

В браузере работают несколько подсистем:

- HTML Parser → DOM tree
- CSS Parser → CSSOM
- Rendering Engine → отрисовка
- Network Stack → HTTP/WS запросы
- JavaScript Engine (например, V8)

Среда выполнения взаимодействует с API операционной системы и предоставляет Web APIs (DOM, Fetch, WebSocket, Crypto и т.п.).

## Стандарты

ECMA International — организация, поддерживающая стандарт ECMAScript (спецификация JavaScript).

Некоторые известные стандарты ECMA:

- ECMA-262 — ECMAScript (JavaScript)
- ECMA-376 — Office Open XML
- ECMA-119 — ISO9660

ECMAScript развивается релизами: ES5, ES6 (ES2015), ES2016, ... ES2022 и т.д. Под «ES6+» обычно понимают современные версии языка (ES2015 и новее).

## Этапы загрузки страницы в браузере

1. Создаётся структура HTML‑документа (парсинг).
2. Применяются стили (CSSOM) и формируется дерево рендеринга.
3. Загружается и отображается контент (картинки, медиа).
4. Выполняются скрипты (JavaScript).

Браузер начинает выполнять JS в момент, когда парсер встречает тег <script> (поведение зависит от атрибутов `defer`/`async` и положения тега). Поэтому часто рекомендуют подключать скрипты внизу body или использовать `defer`.

Если в коде возникает ошибка, браузер выводит её в консоль, но обычно продолжает загрузку остального контента.

Код может выполняться и после загрузки страницы (например, обработчики событий, таймеры, промисы и т.д.) — пока страница открыта.

## Этапы выполнения JS‑скриптов

1. Engine (движок):
   - parsing — лексический и синтаксический анализ
   - interpreter — интерпретатор (часто преобразует в байт‑код)
   - JIT compiler — компилятор в машинный код для «горячих» участков
   - garbage collector — сборщик мусора
2. Runtime (среда выполнения):
   - execution context — контекст выполнения
   - event loop — цикл событий
     - call stack — стек вызовов
     - task queue / callback queue — очередь задач
   - Web APIs — дополнительные API среды (DOM, Fetch, WebSocket, Storage, Crypto и т.д.)

## Инструкции (statements)

Инструкция — минимальное завершённое действие в языке, обычно занимает одну строку. Инструкции разделяются `;` или переводом строки.

Пример:

```js
if (a > 0) console.log('a > 0');
else console.log('a <= 0');
```

В большинстве случаев можно опускать `;` при переносе строк, но есть исключения, например когда следующая строка начинается с `(` или `[` — автоматическая вставка `;` (ASI) может привести к неожиданным результатам.

Неправильный пример, приводящий к ошибке:

```js
arr = [1, 2, 3]
[1, 2, 3].forEach(item => console.log(item))
// В некоторых случаях это будет интерпретировано как обращение к previousResult[1,2,3] и вызовет ошибку
```

Лучше оставлять явные `;` в потенциально неоднозначных местах.

## Операторы

### Унарные операторы

Унарные операторы действуют над одним операндом (пробел между оператором и операндом обычно не ставится):

- -a  (унарный минус)
- +a  (унарный плюс — приведение к Number)
- a++ (постфиксный инкремент)
- ++a (префиксный инкремент)
- a--, --a
- a.b (доступ к свойству через точку)
- ...a (rest / spread)
- ~a  (побитовое NOT)
- !a  (логическое НЕ)
- typeof, void, delete, new (эти операторы обычно пишутся с пробелом после)

Обратите внимание: `&`, `|`, `^`, `<<`, `>>` — это бинарные (побитовые) операторы и требуют два операнда.

### Бинарные операторы

Бинарные операторы обычно разделяются пробелами с двух сторон:

- a + b, a - b, a * b, a / b, a % b
- a = b (присваивание)
- a ** b (возведение в степень)
- 'a' + 'b' (конкатенация строк)
- a & b, a | b, a ^ b (побитовые операции)
- a << b, a >> b, a >>> b (сдвиги)
- a == b (нестрогое сравнение)
- a === b (строгое сравнение)
- a != b, a !== b
- a > b, a < b, a >= b, a <= b
- a && b, a || b
- a ?? b (nullish coalescing)
- a ? b : c (тернарный оператор)

### Тернарный оператор

Правильное форматирование помогает избежать путаницы с опциональной цепочкой:

```js
let result = (a === null || a === undefined) ? b : a;
```

### Оператор опциональной цепочки

Оператор опциональной цепочки `?.` позволяет безопасно обращаться к свойствам у потенциально `null/undefined` значений:

```js
const obj = null;
console.log(obj?.key); // undefined, не бросает ошибку
```

### Логические операторы

Логические операторы в JavaScript возвращают сам операнд (не обязательно boolean):

- `a || b` — возвращает первый truthy‑операнд или последний операнд, если все falsy.
  Пример: `null || '' || 100 || 200 // 100` (первое truthy — 100)

- `a && b` — возвращает первый falsy‑операнд или последний операнд, если все truthy.
  Пример: `'Hello' && 100 && null && 300 // null` (первый falsy — null)

- `!a` — логическое НЕ; двойное отрицание приводит значение к boolean: `!!value`.

- `a ?? b` — nullish coalescing: возвращает `a`, если `a` !== null && a !== undefined, иначе `b`.
  Отличается от `||` тем, что `''` и `0` считаются допустимыми значениями (не заменяются).

Примеры:

```js
let a = null;
let b = 'default';
console.log(a ?? b); // 'default'
console.log('' ?? b); // ''  (в отличие от '' || b -> 'default')
```

### Приоритет операторов (упрощённо)

1. `()`
2. `.` `[]`
3. `new`
4. `++` `--`
5. унарные `!`, `~`, `+`, `-`, `typeof`, `void` и т.д.
6. `*` `/` `%`
7. `+` `-`
8. `<<` `>>` `>>>`
9. сравнения `<` `<=` `>` `>=`
10. `==` `!=` `===` `!==`
11. побитовые `&` `^` `|`
12. `&&`
13. `||`
14. `?:`
15. присваивание `=` и варианты
16. `,` (запятая)

(Для точной таблицы приоритетов смотрите спецификацию или MDN.)

## Переменные

В современных версиях языка используют `let` и `const` (ES2015). `var` — старый способ объявления с функциональной областью видимости.

### let

Блочная область видимости:

```js
let age = 10;
if (true) {
  let age = 20;
  console.log(age); // 20
}
console.log(age); // 10
```

`let` можно переопределить (assign), но нельзя переобъявить в той же области:

```js
let age = 10;
age = 20; // OK
// let age = 30; // SyntaxError (повторное объявление)
```

### const

Константа с блочной областью видимости — нельзя переназначить ссылку:

```js
const GAP_SIZE = 10;
// GAP_SIZE = 15; // TypeError: Assignment to constant variable.
```

Для изменяемых типов (массивы, объекты) `const` защищает от переназначения самой переменной, но внутреннее состояние можно изменять:

```js
const arr = [1, 2, 3];
arr.push(4); // OK
// arr = 4; // TypeError
```

### var

`var` имеет функциональную (не блочную) область видимости и подвержен hoisting:

```js
function test() {
  if (true) {
    var a = 10;
  }
  console.log(a); // 10 — видно за пределами блока if
}
```

Можно переобъявлять `var` в той же области:

```js
var a = 10;
var a = 20; // OK
```

При объявлении `var` в глобальной области на браузерном окружении создаётся свойство глобального объекта `window`.

## Префиксная и постфиксная инкрементация

Префикс:

```js
let a = 10;
let b = ++a; // a = 11, b = 11
```

Постфикс:

```js
let a = 10;
let b = a++; // a = 11, b = 10
```

## Hoisting (всплытие)

Hoisting — поведение JavaScript, при котором объявления переменных и функций поднимаются вверх области видимости. Для `var` это означает, что переменная объявлена (но не инициализирована) до строки объявления и имеет значение `undefined` до присваивания.

```js
console.log(age); // undefined
var age = 10;
```

Для `let` и `const` доступ до объявления вызывает ReferenceError (temporal dead zone):

```js
console.log(name); // ReferenceError
let name = 'Ivan';
```

## Типы данных

В JavaScript есть примитивные типы и объекты. К примитивам относятся:

1. string
2. number (включая Infinity и NaN)
3. bigint
4. boolean
5. symbol
6. undefined
7. null

`object` — для объектов, массивов, функций (функция — это callable object).

Примитивы не являются объектами, но при вызове методов (например, `"text".toUpperCase()`) движок временно оборачивает примитив в объект‑обёртку.

### string

Строки могут быть в одинарных, двойных или обратных кавычках:

```js
let name = "Ivan";
let lastName = 'Petrov';
let fullName = `Ivan Petrov`;
```

Template literals (шаблонные литералы) в обратных кавычках поддерживают интерполяцию `${}`.

### number

Числовой тип покрывает целые и дробные числа, а также специальные значения:

```js
let int = 42;
let float = 3.14;
let bin = 0b1010;
let oct = 0o12;
let hex = 0xA;
let exp = 1.5e2;
```

Специальные значения: `Infinity`, `-Infinity`, `NaN`.

Пример с NaN:

```js
5 - '5'   // 0  (строка приводится к числу)
5 - 'a'   // NaN
Number.isNaN(5 - 'a') // true
isNaN('a') // true (старое поведение: сначала пытается привести к Number)
```

Методы числа:

- toFixed, toExponential, toLocaleString, toPrecision, toString, valueOf

### bigint

Для очень больших целых чисел используется `bigint` (суффикс `n`):

```js
let big = 9007199254740991n;
console.log(typeof big); // 'bigint'
```

BigInt не смешивается с Number в арифметике — выражение `1n + 1` вызовет TypeError; нужно явно приводить типы.

### boolean

Логический тип: `true` или `false`.

### Symbol

Уникальные идентификаторы:

```js
let id1 = Symbol('test');
let id2 = Symbol('test');
console.log(id1 === id2); // false
```

### undefined и null

`undefined` — значение по умолчанию при объявлении переменной без инициализации.

`null` — намеренно заданное «пустое» значение. Исторически `typeof null` возвращает `'object'` — это известная особенность языка.

### object

Объекты — наборы пар ключ:значение. Доступ к свойствам через `.` или `[]`:

```js
let person = { name: 'Ivan', age: 10 };
console.log(person.name); // Ivan
console.log(person['name']); // Ivan
```

#### array

Массивы — упорядоченные коллекции значений:

```js
let arr = [1, 2, 3];
console.log(arr[0]); // 1
```

#### function

Функция — callable object, может храниться в переменной или передаваться как аргумент:

```js
const add = function(a, b) {
  return a + b;
};
console.log(add.toString());
```

## Частые замечания и советы

- Используйте `===` вместо `==`, чтобы избегать неочевидных преобразований типов.
- Для проверки NaN используйте `Number.isNaN`, а не глобальную `isNaN` (последняя приводит аргумент к Number).
- Понимайте отличия между `||` и `??` при выборе значений по умолчанию.
- Будьте осторожны с автоматической вставкой `;` (ASI) в JavaScript.
